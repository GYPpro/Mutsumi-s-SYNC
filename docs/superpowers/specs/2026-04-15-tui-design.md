# Mutsumi's SYNC 窗口化 TUI 设计文档

> 更新日期：2026-04-15

## 1. 概述

将现有的流式日志 TUI 重构为窗口化 TUI，实现多对话管理、实时状态监控和交互式详情查看。

## 2. 技术选型

- **框架**: Textual（现代 Python TUI 框架，异步友好）
- **日志**: 写入文件（debug 级别），TUI 仅显示内存消息

## 4. 功能规格

### 4.1 对话管理

| 功能 | 描述 |
|------|------|
| 用户切换 | ← → 键在不同用户/对话实例间切换 |
| 轮次切换 | ↑ ↓ 键在同一用户的不同对话轮次间切换 |
| 详情查看 | Enter 键进入当前对话的详细视图 |
| 返回 | Esc 键从详情视图返回列表 |

### 4.2 状态栏

| 状态项 | 检测方式 |
|--------|----------|
| AI API | 定时 ping DeepSeek API |
| NapCat | WebSocket 连接状态 |
| 消息计数 | 累计接收的消息数量 |

### 4.3 日志处理

- **日志级别**: DEBUG 级别写入日志文件
- **日志格式**: `时间戳 [级别] 模块: 消息内容`
- **日志文件**: `./logs/mutsumi.log`（每日轮转）

## 5. 数据结构

### 5.1 对话实例

```python
class Conversation:
    peer_id: str          # 用户 ID 或群 ID
    peer_name: str       # 用户昵称或群名称
    chat_type: int       # 1=私聊, 2=群聊
    rounds: list[Round]  # 对话轮次列表
    message_count: int   # 消息计数


class Round:
    timestamp: str        # 时间戳
    user_message: str    # 用户消息
    bot_message: str     # Bot 回复
    tool_calls: list[ToolCall]  # 工具调用列表
    tool_results: list[str]      # 工具返回结果
```

### 5.2 状态数据

```python
class Status:
    ai_api_online: bool   # AI API 是否在线
    napcat_online: bool  # NapCat 是否在线
    message_count: int   # 总消息数
```

## 6. 交互流程

1. **启动阶段**
   - 初始化 Textual 应用
   - 检查 API 可用性
   - 清空屏幕，显示主界面

2. **运行时**
   - 实时接收消息，更新当前对话
   - 定时检查 API 状态
   - 用户通过键盘交互切换视图

3. **用户交互**
   - 监听键盘事件
   - 根据当前视图处理不同操作
